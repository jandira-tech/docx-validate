# Release checklist

`bun run release` will read this file, prompt `[y/N]` for each unchecked item
below, and abort if any answer is "no". Edit this file to add or remove
items the way the project actually works — the wrapper does not hard-code
the list.

Format: only top-level `- [ ] …` lines are treated as checklist items. Use
`- [x]` to permanently mark an item as "always satisfied" (the wrapper
won't prompt for those). Headers, prose, and indented bullets are ignored.

## Pre-release

- [ ] CHANGELOG.md updated with a new section for the version being released
- [ ] README.md reflects the current public API surface (programmatic-use snippets, install instructions)
- [ ] llms.txt reflects the current public API surface (kept in sync with README)
- [ ] CLAUDE.md behavioural notes still match the code
- [ ] SECURITY.md still accurate if anything in the security surface changed (LD_PRELOAD shim, native code paths, env var names)
- [ ] JSDoc on changed exports is up to date (renamed parameters, new options)

## Verification

- [ ] `bun run barrel:check` — no drift between source exports and `src/index.ts`
- [ ] `bun run test` — all green
- [ ] `bun run lint` — 0 warnings, 0 errors
- [ ] `bun run fmt` — no formatter drift
- [ ] `bun run cspell` — no unknown words
- [ ] `bun run build` — `dist/index.mjs` builds cleanly
- [ ] `dist/index.mjs` scanned for unexpected patterns (`grep -E "LD_PRELOAD|SHIM_SOURCE|RTLD_NEXT|dlsym" dist/index.mjs` returns no matches)
- [ ] Lockfile (`bun.lock`) committed if `package.json` deps changed

## Migration / breaking changes

- [ ] If breaking: MIGRATION.md updated (skip for patch / minor non-breaking releases)
- [ ] If breaking: deprecation warnings added in the prior minor release
